管理員操作手冊

LINE Bot 貸款案件管理系統 — Web 管理介面

一、登入系統

網址：https://loan-system-a7xd.onrender.com/login
角色可用功能
管理員全部功能（群組管理、密碼管理、操作紀錄、下載備份、清除資料、日報改進度）
行政B行政B作業、日報、查詢、客戶資料庫、新增客戶
一般查詢日報、查詢、新增客戶、客戶資料庫
業務群只看自己群組的日報 + 客戶資料庫 + 新增客戶
多台電腦、多個分頁可以同時登入，密碼共用不綁裝置。同一瀏覽器只能登一個帳號。

二、Web 各頁面說明

日報 /report

  • 查看各群組客戶狀態（點群組名稱展開）
  • 統計卡片：新客戶/補件/送件/未對保/本月結案/今日進件/待撥款
  • 匯出 CSV — 下載客戶清單
  • 批次結案 — 勾選客戶 → 右下角按鈕
  • 修改進度（管理員專用）— 展開客戶 → 輸入框 → 儲存

查詢 /search

  • 搜姓名/身分證，篩選群組/日期
  • ✏️ 編輯 → 跳到修改客戶資料
  • 操作歷程（管理員專用）— 展開看完整 case_logs

歷史 /history

  • 已結案客戶（結案/違約金/放棄/拒絕）

客戶資料庫 /pending-customers

  • 搜尋、篩選、分頁（每頁 50 筆）
  • ✏️ 編輯 → 修改所有欄位

新增客戶 /new-customer

  • 手動填客戶資料表單
  • 支援身分證、居留證格式（A123456789 / A812345678 / AC12345678）
  • 負債明細可編輯/新增/刪除

行政B作業 /adminb

  • 搜尋客戶 → 方案判別 → 規則校驗 → 下載申請書 Excel
  • 自動調整規則（年資/月薪/居住/學歷）
  • 勾選方案時可看到各方案金額期數

三、管理員專屬功能

群組管理 /admin/groups

新增群組步驟：
1. LINE 群組打 @AI 群組ID
2. 網頁點「+ 新增群組」
3. 貼 ID、取名、選類型
4. 到「密碼管理」設密碼
群組類型用途
業務群業務員建立客戶、貼進度
行政群接收排撥名單
A群行政回貼核准/婉拒、撥款名單

密碼管理 /admin/passwords

管理員密碼全部權限
行政B密碼行政B作業
一般查詢密碼只讀查詢
各群組密碼業務用群組身份登入

操作紀錄 /admin/logs

查看所有 case_logs，可搜尋、分頁。

下載備份 /admin/download-db

下載 SQLite 資料庫檔，建議定期備份（每週或每月）。

清除資料 /admin/reset_data

危險操作！清除所有案件資料，群組設定保留。
正式上線後請勿使用！僅供測試階段清除測試資料。

四、LINE Bot 指令規則

訊息格式

姓名 + 公司 + 狀態 + 備註（備註多長都可以）

唯一規則：姓名和公司之間一定要有空格
其他地方可以不空格，全形半形都可以。

核准/婉拒判定

判定關鍵字
核准（優先）核准、核準
婉拒婉拒、申覆失敗、建議維持原審、不予承作、無法再進件、無法承作、30日內有進件、已建檔、不提供申覆
已補照會補照會、照會
已補申覆補申覆、申覆
已補資料補件、補資料、補行照、補聯徵、補保人、補薪轉、補照片、補時段
NANA、未接照會
同時有核准+婉拒關鍵字 → 以核准為準

五、訊息流程

業務群貼民間方案不推 A 群

銀行/零卡/房地/代書/當舖/鄉民/新鑫等 → 只記錄到日報，不推 A 群

送件順序 → 日報區塊

  • 貸款方案（亞太/和裕/21等） → 先在「送件」區塊，A群回貼後移到公司區塊
  • 民間方案（銀行/房地等） → 直接顯示在對應區塊
  • 同時送件（照會+） → 多區塊顯示，各自獨立狀態
  • 核准 → 移到「待撥款」區塊
  • 婉拒 → 從日報移除（自動推下一家）

公司不在路線會跳按鈕

按鈕效果
再送XX加到送件順序，設為當前公司（有二次確認）
同送XX加到同時送件清單（有二次確認）
取消不處理

八、正式上線步驟

1. 新增群組
在每個 LINE 群組打 @AI 群組ID → 網頁「群組管理」新增

2. 設密碼
到「密碼管理」設各群組+各角色密碼

3. 測試
打 @AI 群組ID 確認 → 貼測試客戶 → 打 @AI 日報 確認

4. 發手冊給業務和行政
業務群操作手冊 / A群操作手冊
