LINE Bot 貸款案件管理系統

使用手冊 — 業務 / 行政人員適用

一、基本規則

觸發方式

所有指令都需要加上 @AI（或 @ai），系統才會處理。一般聊天不會觸發。

例外：建立新客戶和設定送件順序不需要 @AI，系統會自動辨識格式。

需要 @AI：王小明 結案 @AI
需要 @AI：@AI 查 王小明
需要 @AI：王小明 補件 @AI（補件回貼A群）
不需要 @AI：04/13-王小明-A123456789（建立客戶，自動辨識）
不會觸發：王小明結案了喔（一般聊天，沒有 @AI）

姓名規則

  • 姓名支援 2~6 個中文字（含原住民姓名）
  • 超過 6 個字：先用短名建立，再用 @AI 改名 修正
  • 姓名和其他內容之間建議有空格，避免系統誤判
  • 打錯名字系統會找不到客戶，可用 @AI 舊名 改名 新名 修正

錯誤回報

所有指令如果執行失敗，系統會自動回覆錯誤訊息：

❌ 指令執行失敗：OperationalError: database is locked

看到這種訊息代表系統有問題，請截圖回報給管理員處理。正常使用不會出現。

二、業務群操作

建立新客戶（不需要 @AI）

直接在群組貼客戶資料，格式：

04/13-王小明-A123456789
電話：0912345678
公司：XX有限公司
月薪：45000
第一行必須包含：日期-姓名-身分證。系統會自動建立客戶到日報。
建立客戶不會回貼到 A 群。

設定送件順序（不需要 @AI）

04/13-王小明-亞太/和裕/21

用斜線 / 分隔公司名稱，系統會按順序送件。

補件 / 進度更新（需要 @AI）

補件或進度更新必須加 @AI，系統才會處理並回貼到 A 群：

王小明 補照會時段14:00-15:00 @AI
王小明 補薪轉 @AI
如果不加 @AI，系統不會處理，也不會回貼到 A 群。
這是為了避免把一般聊天誤當成補件。

三、@AI 指令大全

查詢類

指令說明
@AI 日報產生本群組的日報
@AI 查 王小明查詢客戶詳細資訊
@AI 群組ID顯示此群組的 ID（設定用）
@AI 統計今日/本月 進件、核准、結案數量
@AI 待撥款列出所有核准待撥款客戶

送件操作

指令說明
@AI 王小明 轉下一家推進到送件順序的下一家
@AI 王小明 轉和裕直接跳到指定公司
@AI 王小明 婉拒目前公司婉拒，自動推下一家
@AI 王小明 婉拒轉21婉拒後跳到指定公司

結案操作

指令說明
@AI 王小明 結案結案，從日報移除
@AI 王小明 結案 客戶取消結案並記錄原因
@AI 王小明 違約金已支付5000違約金結案
@AI 王小明 重啟已結案客戶恢復到日報

修改資料

指令說明
@AI 王小明 改名 王曉明修正客戶姓名（也能改已結案的）
@AI 王小明 改身分證 A123456789修正身分證號碼
@AI 王小明 亞太商品 核准 12萬修正核准金額
修改金額時，姓名和公司之間一定要有空格：
正確：@AI 王小明 亞太商品 核准 12萬
錯誤：@AI 王小明亞太商品核准12萬（名字會抓錯）

批次操作

指令說明

王小明

@AI 批次婉拒

第一行打指令，下面每行一個姓名。系統會逐一處理並回報結果。

照會

指令說明

@AI 王小明 照會同上（前後都可以）
@AI 照會 王小明 21指定公司的照會

四、A 群（行政群）操作

回貼進度

A 群貼客戶進度（核准、婉拒、補件等），系統會自動回貼到對應的業務群：

王小明
亞太核准12萬
同名客戶如果有多位，系統會跳出按鈕讓你選擇。

撥款名單

貼撥款名單格式，系統自動批次更新撥款日期：

04/13 21 撥款名單
王小明
李大華

五、日報說明

  • 日報顯示的日期是客戶建立日期，不是當天日期
  • 今日新建的客戶前面會有 🆕 標記
  • 多家核准會各自顯示撥款狀態：
    核准和裕5萬(撥款04/10)/亞太12萬(待撥款)
  • 待撥款超過 7 天的客戶，日報最後會獨立提醒
  • 日報分三段：貸款方案 → 民間方案 → 房地
    每段只在有客戶時才顯示

七、注意事項

常見錯誤與解法：
  • 系統沒反應 → 檢查有沒有加 @AI
  • 找不到客戶 → 檢查姓名有沒有打錯，可用 @AI 查 姓名 確認
  • 金額修改出錯 → 姓名和公司之間一定要有空格
  • 同名客戶 → 系統會跳按鈕讓你選哪一個
  • 客戶被誤結案 → 用 @AI 姓名 重啟 恢復
  • 名字打錯 → 用 @AI 舊名 改名 新名 修正
  • 出現 ❌ 錯誤訊息 → 截圖回報管理員
重要提醒：
  • 建立客戶不會回貼到 A 群，只記錄到日報
  • 補件更新必須加 @AI 才會回貼到 A 群
  • 一般聊天不會被系統處理，放心聊天
  • 密碼請找管理員修改（網站 → 密碼管理）
